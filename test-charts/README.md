# Refinement test charts

Optional charts for pushing the calibration further than the standard color/B&W
charts. **Print every one with the correction turned OFF / strength 0** — these
measure the *raw* film, not the corrected output. Scan each test single-pass with
the scanner's auto-exposure and auto-color **off**.

Generated by `drawGridChart` / `drawFlatField` in [../js/chart.js](../js/chart.js).
All except the vignette field keep the 74% centered safe area so the Lab's phone
alignment can't clip the fiducials.

| File | What it measures | How to shoot it |
|------|------------------|-----------------|
| `test-vignette-field.png` | **Lens vignetting** — how brightness/color falls off from center to corners. Fills the whole frame (no safe area) so the corners are measurable; only tiny corner fiducials. | Print once. AE-independent (falloff is relative), so it's one of the few things we can pin down reliably → becomes a spatial gain-map correction. |
| `test-color-cube-1.png` + `test-color-cube-2.png` | **Saturated-color accuracy** — a 6×6×6 (216-patch) sweep of the RGB cube, split across two charts (108 each). Feeds a future 3D interpolation solver to replace the per-channel model. | Print **both**. |
| `test-extreme-tone.png` | **Toe & shoulder** — dense sampling of deep shadows (0–20%) and bright highlights (80–100%), where the crush and clip actually happen. | Print once. |
| `test-repeatability.png` | **Frame-to-frame variance** — 16 large, easy-to-read patches. Doesn't refine the correction; tells us how repeatable the Lab even *is* (so we know when to stop chasing). | Print **3–5 copies**, then scan them all together in one pass. |

When you've scanned any of these, drop them in and the matching extractor + solver
gets built (vignette → spatial correction; cube → 3D color; etc.).
